## Diagnostic Summary

The 5 new flow files (onboarding, exception, template, gate-model, visual) were generated by subagents during the portal refactoring. While the 134 existing E2E checks all pass, the new flows have pervasive bugs caused by selectors that don't match actual template HTML.

## Rewrite Strategy

### Pattern: Use Body Text Assertions (like scope_flow.py)

The passing `scope_flow.py` uses simple body text checks rather than fragile CSS selectors:
```python
body = (await self.page.text_content("body") or "").lower()
assert "850-beg-01" in body, "Expected error code not visible"
```

For the new flows, prefer this pattern for verification steps. Use CSS selectors only for interactions (filling forms, clicking buttons).

### Pattern: Self-Contained Step Groups

Each multi-step operation (e.g., "fill step 2 form + submit") should be a single test step that:
1. Navigates to `/onboarding` (or the target page)
2. Performs all interactions (fill + click)
3. Waits for navigation (`expect_navigation`)
4. Navigates back to verify state

### Pattern: Cross-Portal Login/Logout

For flows that switch portals (exception, template):
```python
# Logout from current portal
await self.page.goto(f"{current_url}/logout", wait_until="domcontentloaded")
# Login to new portal
await self.page.goto(f"{new_url}/login", wait_until="domcontentloaded")
await self.page.fill('input[name="username"]', username)
await self.page.fill('input[name="password"]', password)
async with self.page.expect_navigation():
    await self.page.click('button[type="submit"]')
```

## Ralph Loop Rules (from orchestrator prompt)

1. **Never skip testing.** Every flow rewrite ends with a Playwright run.
2. **Append-only feedback.** Every failure logged to `playwrightcli/feedback.md`.
3. **Tiered correction:**
   - Iteration 1: Fix specific failing selector based on error output
   - Iteration 2: Re-read template HTML, compare to flow, fix structural issues
   - Iteration 3: Simplify — use body text assertion instead of CSS selector
4. **Max 3 iterations per flow.** Document remaining failures and move on.
5. **Gate between flows.** Flow N+1 doesn't start until Flow N's loop exits.
